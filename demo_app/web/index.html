IBM Security Verify Integration - Test Application

The integration requires configuration of a JWT authentication module and a BA header from IBM Application Gateway or WebSEAL to authenticate to provide identity information. When the integration is configured correctly, identity information is injected into the incoming request.

The pages below are for testing the various <auth-constraint> elements in the web.xml file of a bundled Java Web Application (WEB-INF directory). To access the pages, ensure you have defined the roles below either by configuring groups-to-roles mapping in the JWT authentication module.

Who am I?
This page reads identity information provided by the Java Principal object available in the HTTP Servlet Request. These object should reflect the attributes set by the JWT authentication module being used. This should show the Username attribute and Groups that a user is a member of.

Security roles test page
A user will not be permitted to access this page if they do not have the "developer" security-role. This should be a group membership from the identity source that is mapped via the authentication module to Java roles

Step-up authentication demonstration page
This page can be used to test the step-up authentication capabilities of IBM Application Gateway or IBM Security Verify Access. For this scenario to work the step-up authentication scenario must be configured.

Logout page
This page will invalidate the user's session then redirect to WebSEAL's logout page to complete the logout process.
